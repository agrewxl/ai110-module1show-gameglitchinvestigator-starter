# 🎮 Game Glitch Investigator: The Impossible Guesser

## 🚨 The Situation

You asked an AI to build a simple "Number Guessing Game" using Streamlit.
It wrote the code, ran away, and now the game is unplayable. 

- You can't win.
- The hints lie to you.
- The secret number seems to have commitment issues.

## 🛠️ Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Run the broken app: `python -m streamlit run app.py`

## 🕵️‍♂️ Your Mission

1. **Play the game.** Open the "Developer Debug Info" tab in the app to see the secret number. Try to win.
2. **Find the State Bug.** Why does the secret number change every time you click "Submit"? Ask ChatGPT: *"How do I keep a variable from resetting in Streamlit when I click a button?"*
3. **Fix the Logic.** The hints ("Higher/Lower") are wrong. Fix them.
4. **Refactor & Test.** - Move the logic into `logic_utils.py`.
   - Run `pytest` in your terminal.
   - Keep fixing until all tests pass!

## 📝 Document Your Experience

**Purpose of the game.** A simple Streamlit number-guessing game: the app picks a secret number inside a range that depends on the chosen difficulty (Easy 1–20, Normal 1–100, Hard 1–50), and the player guesses until they find it or run out of attempts. After each guess the game gives a "higher/lower" hint and updates a score.

**Bugs I found** (full details and reproduction logs in [reflection.md](reflection.md)):
- **Impossible to win / "secret changes its mind."** On every even attempt `app.py` converted the secret to a string, so an int guess was compared to a string. This raised a silently-swallowed `TypeError` and fell back to *lexicographic* comparison — e.g. 27 said "go higher" and 28 said "go lower," with no value that ever won.
- **Backwards hints.** A guess that was *too high* displayed "📈 Go HIGHER!"; the hint pointed the player the wrong way.
- **"New Game" was broken.** It only reset `attempts`, leaving `status`, `score`, and `history` stale, so after losing you stayed locked on "Game over." It also ignored the difficulty range.
- **Wrong range text + off-by-one attempt counter.** The prompt always read "between 1 and 100," and attempts started at 1 instead of 0.

**Fixes I applied:**
- Refactored the core logic (`get_range_for_difficulty`, `parse_guess`, `check_guess`, `update_score`, plus a new `message_for_outcome`) out of `app.py` and into [logic_utils.py](logic_utils.py).
- `check_guess` now coerces both values to `int`, so comparisons are always numeric. Removed the string-conversion in `app.py`.
- Hints are now generated by `message_for_outcome`, which maps "Too High" → "Go LOWER!" and "Too Low" → "Go HIGHER!".
- Replaced the broken reset with a single `start_new_game()` helper that resets *all* state and uses the correct difficulty range.
- Fixed the prompt to show the real range and corrected the attempt counter.
- Added two regression tests so the bugs can't silently return.

## 📸 Demo Walkthrough

A sample game on **Normal** difficulty (range 1–100, 8 attempts) where the secret is **63**:

1. The app loads showing "Guess a number between 1 and 100. Attempts left: 8" and a score of 0.
2. User enters a guess of **50** → game shows "📈 Go HIGHER!" (Too Low). Score becomes **-5**, attempts left: 7.
3. User enters a guess of **75** → game shows "📉 Go LOWER!" (Too High). Score becomes **-10**, attempts left: 6.
4. User enters a guess of **63** → game shows "🎉 Correct!", triggers balloons, and reveals "You won! The secret was 63." Score jumps by `max(10, 100 − 10×attempts)` = **70**, for a final score of **60**.
5. The board locks with "You already won. Start a new game to play again."
6. User clicks **New Game 🔁** → a fresh secret is rolled within range and score, attempts, status, and history are all reset to a clean slate.
